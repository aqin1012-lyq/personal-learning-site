# Screenshot Shotlist

Use this when capturing the first public screenshots for README and launch posts.

## Shot 1: Homepage hero
Purpose:
- show the visual tone
- show this is not a generic blog

## Shot 2: Homepage positioning block
Purpose:
- explain Logs / Notes / Projects quickly
- show template onboarding value

## Shot 3: Timeline explorer
Purpose:
- highlight the strongest visual differentiator
- show long-term learning rhythm

## Shot 4: Logs page
Purpose:
- show process-oriented writing and filtering

## Shot 5: Notes page
Purpose:
- show reusable knowledge cards

## Shot 6: Projects page
Purpose:
- show output validation

## Shot 7: Studio page
Purpose:
- show that the template is operable, not just pretty

## Optional detail shots
- note detail page
- log detail page
- project detail page
- about page

## Capture rules
- use coherent content, not obviously placeholder screens
- avoid showing broken or sparse sections unless demonstrating empty state intentionally
- make sure domain, author, and title look intentional
- prefer a consistent browser width and clean framing
