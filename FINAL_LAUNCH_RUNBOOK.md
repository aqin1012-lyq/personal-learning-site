# Final Launch Runbook

Use this file on launch day.

## T-1: before publishing

- [ ] confirm the public demo URL works
- [ ] confirm `siteUrl` uses the real domain
- [ ] confirm README is up to date
- [ ] confirm homepage, logs, notes, projects, and studio all render correctly
- [ ] confirm release notes and GitHub release text are ready
- [ ] confirm screenshots are exported and easy to attach

## T-0: GitHub prep

- [ ] update repo description
- [ ] update repo topics
- [ ] upload or reference repo social card
- [ ] create the first GitHub release using `GITHUB_RELEASE_TEMPLATE.md`
- [ ] pin demo URL in README if available

## T+0: first public post

Choose one primary channel first.

Recommended order:
1. GitHub live
2. primary post on X / 即刻 / 小红书
3. longer explanation thread or article
4. invite feedback and examples

## Suggested launch framing

Lead with:
- this is a personal learning system template
- it connects logs, notes, and projects
- it helps long-term learners build a public learning interface

Do not lead with:
- Next.js version
- Tailwind version
- technical architecture details

## T+1h: immediate checks

- [ ] can users open the demo?
- [ ] does the social preview image render?
- [ ] do users understand what the product is in under 10 seconds?
- [ ] are there repeated questions showing onboarding confusion?

## T+24h: post-launch review

- [ ] collect the first user questions
- [ ] note where users got confused
- [ ] update README quickly if the same question appears twice
- [ ] update launch copy if people misunderstand the product positioning
- [ ] save useful feedback for v0.2 planning
