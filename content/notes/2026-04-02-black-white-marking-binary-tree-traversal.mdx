<p>可是迭代法就没那么简单了&#xff0c;我第一次写的时候三种遍历除了都用了while循环都用了栈结构&#xff0c;其他的逻辑都是重新捋的&#xff0c;并不像&#x1f446;迭代法一样&#xff0c;写出一个&#xff0c;另外两种遍历改下顺序就成&#61; &#61;&#xff0c;But不应该啊&#x1f914;&#x1f914;&#x1f914;&#x1f914;&#x1f914;&#x1f914;&#x1f914;</p>

<p>为什么递归可以&#xff0c;迭代不行&#xff1f;</p>

<p>二刷的时候又重新捋了捋这二叉树的遍历逻辑&#xff0c;发现其实迭代法之所以看起来如此简洁&#xff0c;是因为由于递归中方法不断调用调用&#xff0c;系统为我们隐性的维护了一个“栈”&#xff0c;而在迭代法中&#xff0c;我们只能自己来维护这个“栈”&#xff0c;而前序/中序/后序遍历之所以无法统一的主要原因在于<strong>处理节点的顺序和访问节点的顺序是不一致的</strong>&#xff08;感谢卡哥的代码随想录&#xff09;&#xff0c;简单来说就是无法确定当前访问的节点是不是已经访问过了。</p>

<p>卡哥在代码随想录中提供了一种适用于三种遍历方式的统一写法&#xff0c;用这种统一的写法&#xff0c;先序遍历、中序遍历、后序遍历也像递归法一样只需要进行位置上的简单调整即可&#xff0c;这种统一写法的基本原理就是在处理过的节点再次入栈时&#xff0c;后面紧跟着添加一个“null”&#xff0c;当遍历到“null”时&#xff0c;说明下一个节点已经访问过了&#xff0c;直接添加到结果集即可。</p>

<p>很机智的方法&#xff0c;很好的解决了“无法确定当前访问的节点是不是已经处理过了”这个问题&#xff0c;但其实在看代码的时候&#xff0c;还是有些不是很好理解的感觉&#xff0c;在整理笔记时&#xff0c;不经意看到了之前学习JVM GC相关的笔记&#xff0c;忽然看到——“三色标记法”&#xff5e;</p>

<p>简单介绍下垃圾回收的三色标记法&#xff0c;以免有不了解的朋友&#xff08;知道的朋友可以直接跳过这part&#xff5e;&#xff09;</p>

<p>三色标记法是一种垃圾回收算法&#xff0c;即在垃圾回收的过程中使用“白”、“灰”、“黑”三种颜色&#xff0c;分别表示“未被标记的垃圾对象”、“已被标记&#xff0c;但其引用的对象尚未全部被标记的非垃圾对象”和“已被标记&#xff0c;且其引用的对象也全部被标记的非垃圾对象”&#xff0c;以便回收不再使用的对象&#xff0c;释放内存空间。</p>

<p>于是&#xff5e;有了这篇文章&#xff0c;希望能更好的帮助大家理并掌握二叉树的各种遍历(&#xffe3;∇&#xffe3;)/</p>

<p>我们就姑且叫它“黑白标记法”&#xff0c;即使用“白色&#xff08;white&#xff09;”和“黑色&#xff08;black&#xff09;”两种颜色来对入栈的节点进行标记&#xff0c;“白色”表示改节点未访问&#xff0c;“黑色”表示该节点已访问&#xff0c;基本逻辑就是在遍历的过程中&#xff0c;碰到“白色”的节点则继续对其子节点进行访问&#xff0c;碰到“黑色”则直接进行逻辑处理&#xff08;这里的处理逻辑比较简单&#xff0c;就是添加进结果集&#xff0c;当然以后碰到更复杂的处理逻辑也是适用的&#xff09;</p>

<p>基于上面的基本思想&#xff0c;我们可以在Leetcode官方提供的TreeNode结构的基础上&#xff0c;再“包”一层</p>

<pre>
<code>//方定义的二叉树结构
public class TreeNode {
      int val;
      TreeNode left;
      TreeNode right;
      TreeNode() {}
      TreeNode(int val) { this.val &#61; val; }
      TreeNode(int val, TreeNode left, TreeNode right) {
          this.val &#61; val;
          this.left &#61; left;
          this.right &#61; right;
      }
}
// 再“包”一层
public class MyTreeNode {
    // white 未处理 black 已处理
      String type;
      TreeNode node;
      MyTreeNode() {}
      MyTreeNode(String type, TreeNode node) {
        this.type &#61; type;
        this.node &#61; node;
      }
}</code></pre>

<p>基于上面构建的MyTreeNode&#xff0c;我们开始遍历&#xff01;</p>

<h2>前序遍历</h2>

<p><img src="/images/notes/2026-04-02-black-white-marking-binary-tree-traversal/04.png" alt="2026-04-02-black-white-marking-binary-tree-traversal-4" /></p>
</p>

<pre>
<code>class Solution {
    public List&lt;Integer&gt; preorderTraversal(TreeNode root) {
        List&lt;Integer&gt; result &#61; new ArrayList&lt;&gt;();
        Stack&lt;MyTreeNode&gt; stack&#61;new Stack&lt;&gt;();
        stack.add(new MyTreeNode(&#34;white&#34;,root));
        while(!stack.isEmpty()){
            MyTreeNode cur &#61; stack.pop();
            if(cur.node&#61;&#61;null){
                continue;
            }

            }
        }
        return result;
    }
}</code></pre>
