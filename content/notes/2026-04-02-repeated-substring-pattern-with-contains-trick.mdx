## 这道题为什么会让人一眼觉得像“脑筋急转弯”

因为它最出名的解法通常特别短：

```java
return (s + s).substring(1, (s + s).length() - 1).contains(s);
```

第一次看到时，很多人的反应都一样：

- 这也行？
- 为什么拼接两次再 contains 一下就能判断？
- 这到底是技巧，还是有严格逻辑？

其实它不是玄学，而是利用了“周期串在旋转后仍然能对齐”的性质。

## 先说问题本身

题目想判断的是：
一个字符串 `s`，是否可以由某个更短的子串重复多次构成。

比如：

- `abab` 可以，由 `ab` 重复两次
- `abcabcabc` 可以，由 `abc` 重复三次
- `aba` 不可以

## 为什么要看 `s + s`

把一个字符串复制并拼接一次，相当于把它的各种“循环位移”都放到了同一个大串里。

举个例子：

`abab + abab = abababab`

如果原串本身有重复周期，那么你把它切开再错位拼上，往往仍然能在中间重新找到原串。

而如果原串没有这种周期结构，它就不会在这种错位环境下自然重新出现。

## 为什么还要去掉头尾

如果直接在 `s + s` 里找 `s`，那当然永远能找到：

- 开头本来就是一个 `s`
- 结尾也可能再出现一个 `s`

这没有任何判断价值。

所以要做的是：
把首尾各去掉一个字符，强行排除“原地匹配”这种废话情况。

也就是说，我们真正问的是：

**除了原本那次位置完全一样的出现之外，`s` 能不能在一个错位后的拼接串里再次出现？**

如果可以，说明它内部大概率有重复节奏。

## 为什么周期串一定能被找到

假设 `s = abcabc`，它由 `abc` 重复两次组成。

那么在 `s + s = abcabcabcabc` 里，你把开头和结尾裁掉一层后，中间依然存在很多错位片段。
由于 `s` 本身有周期结构，这些错位片段里仍然可能重新拼出完整的 `s`。

换句话说：
周期串在“旋转后”仍有机会和自己对齐。

这就是这个技巧成立的底层原因。

## 为什么非周期串不行

如果一个字符串内部没有重复周期，那它的字符排列更像一次性的。

即便你把它复制拼接，再尝试在中间错位区域里找自己，也通常拼不回原串。

因为它没有那个“旋转后还能重新贴合”的内部结构。

## 这个解法真正值得学的点

不是背下某句代码，而是学会一种观察方式：

**当一个对象具有周期性时，把它复制、平移、旋转，常常会暴露出这种周期结构。**

字符串题里这是 `s + s`。
在别的问题里，也可能是数组循环、环形结构、模运算等。

这类思路背后是同一种数学直觉。

## 写题时要注意什么

如果只是为了面试或刷题，这个写法非常漂亮。
但如果是在解释给别人听，最好别只丢出一行代码。

因为一行代码本身不难，难的是理解：

- 为什么拼接两次
- 为什么删掉头尾
- 为什么 contains 能代表周期存在

能把这三层讲清楚，才算真的掌握。

## 最后一段总结

`(s + s).contains(s)` 相关技巧之所以成立，不是因为字符串 API 很神，而是因为：

- 周期串在自拼接后的错位空间里，仍然有机会重新对齐
- 去掉头尾是为了排除原地匹配
- 一旦还能重新出现，就说明它内部大概率由更短模式重复构成

所以这题最值得带走的，其实不是一行代码，而是对“周期性”这件事更敏感的直觉。
