<p>需要解释下什么是LCA吗&#xff1f;</p> 
<p>算了我解释下吧——</p> 
<p><strong>Lowest Common Ancestor</strong>&#x1f449; <strong>最近公共祖先</strong>&#xff0c;意思是两节点在树中<strong>最近的共同父节点</strong>。</p> 
<hr /> 
<h2>先给结论</h2> 
<p><strong>递归返回值如果就是当前问题的最终答案 → 可以直接 return</strong></p> 
<p><strong>递归返回值如果只是“判断材料” → 必须先赋值再处理</strong></p> 
<hr /> 
<h2>递归的两种本质角色</h2> 
<h2>1️⃣ 传话型递归<strong> → </strong>可直接 return</h2> 
<p><strong>特点&#xff1a;</strong></p> 
<ul><li>当前层不加工子结果</li><li>子问题答案 &#61; 当前问题答案</li><li>当前层没有“决策权”</li></ul> 
<p><strong>代码特征&#xff1a;</strong></p> 
<pre><code>return dfs(root.left);</code></pre> 
<h2>2️⃣ 加工型递归<strong> → </strong>必须先赋值</h2> 
<p><strong>特点&#xff1a;</strong></p> 
<ul><li>当前层必须根据左右子结果做判断</li><li>子结果只是“信息”&#xff0c;不是最终答案</li><li>当前节点可能成为答案&#xff08;或者答案的一部分&#xff0c;如<a href="https://leetcode.cn/problems/insert-into-a-binary-search-tree/" title="701. 二叉搜索树中的插入操作" rel="nofollow">701. 二叉搜索树中的插入操作</a>&#xff09;</li></ul> 
<p><strong>代码特征&#xff1a;</strong></p> 
<pre><code>TreeNode left &#61; dfs(root.left);
TreeNode right &#61; dfs(root.right);</code></pre> 
<hr /> 
<h2>我们来看看这两道几乎“长得一样”的 LCA 题</h2> 
<p>附上 leetcode &#x1f517;感兴趣的同学可以先去做一做&#xff0c;然后感触会更深&#xff5e;</p> 
<p><a href="https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-search-tree/" title="https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-search-tree/" rel="nofollow">https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-search-tree/</a></p> 
<p><a href="https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/" title="https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/" rel="nofollow">https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/</a></p> 
<p>为什么拿它俩举例&#xff0c;因为它们<strong>代表上一趴我们提到的两种完全不同的递归模式&#xff1a;传递性递归和加工型递归</strong></p> 
<hr /> 
<h2>235. 二叉搜索树的最近公共祖先&#xff08;BST LCA&#xff09;</h2> 
<h2>1️⃣ 关键前提&#xff1a;BST 的结构性质</h2> 
<p>BST 有一个<strong>强约束</strong>&#xff1a;</p> 
<ul><li>左子树所有节点 &lt; root.val</li><li>右子树所有节点 &gt; root.val</li></ul> 
<p>&#x1f449; <strong>节点值大小&#xff0c;直接决定位置方向</strong></p> 
<h2>2️⃣ 解题思路&#xff08;只走一条路&#xff09;</h2> 
<ul><li>如果 p、q 都小于 root → LCA 在左子树</li><li>如果 p、q 都大于 root → LCA 在右子树</li><li>否则 → root 就是最近公共祖先</li></ul> 
<h2>3️⃣ 递归代码&#xff08;传话型&#xff0c;直接 return&#xff09;</h2> 
<pre><code>public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root &#61;&#61; null) return null;

    if (p.val &lt; root.val &amp;&amp; q.val &lt; root.val)
        return lowestCommonAncestor(root.left, p, q);

    if (p.val &gt; root.val &amp;&amp; q.val &gt; root.val)
        return lowestCommonAncestor(root.right, p, q);

    return root;
}</code></pre> 
<h2>4️⃣ 为什么这里可以直接 return&#xff1f;</h2> 
<p>因为在 BST 中&#xff1a;</p> 
<ul><li>当前节点一旦判断“答案不在我这里”</li><li>就能<strong>确定答案 100% 在某一侧</strong></li><li>当前层不需要处理左右结果</li></ul> 
<p>&#x1f449; <strong>子递归的结果&#xff0c;就是最终答案</strong></p> 
<hr /> 
<h2>236. 二叉树的最近公共祖先&#xff08;普通二叉树&#xff09;</h2> 
<h2>1️⃣ 关键前提&#xff1a;没有任何大小规则</h2> 
<p>普通二叉树中&#xff1a;</p> 
<ul><li><code>left</code> / <code>right</code> 只是结构关系</li><li><code>val</code> 的大小与位置无关</li></ul> 
<p>&#x1f449; <strong>不能靠数值判断方向</strong></p> 
<h2>2️⃣ 解题思路&#xff08;必须左右都找&#xff09;</h2> 
<ul><li>去左子树找 p 或 q</li><li>去右子树找 p 或 q</li><li>根据左右返回结果判断当前节点身份</li></ul> 
<h2>3️⃣ 递归代码&#xff08;加工型&#xff0c;必须赋值&#xff09;</h2> 
<pre><code>public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root &#61;&#61; null) return null;
    if (root &#61;&#61; p || root &#61;&#61; q) return root;

    TreeNode left &#61; lowestCommonAncestor(root.left, p, q);
    TreeNode right &#61; lowestCommonAncestor(root.right, p, q);

    if (left !&#61; null &amp;&amp; right !&#61; null) return root;
    return left !&#61; null ? left : right;
}</code></pre> 
<h2>4️⃣ 为什么这里必须先赋值&#xff1f;</h2> 
<p>因为你要回答的是&#xff1a;</p> 
<p>“p 和 q 是不是分别出现在我左右两棵子树&#xff1f;”</p> 
<p>这个判断<strong>必须同时知道左右子树的搜索结果</strong>&#xff0c; 而当前节点是否是 LCA&#xff0c;取决于<strong>结果的组合状态</strong>。</p> 
<p>&#x1f449; 子递归结果 ≠ 最终答案&#xff0c;只是判断材料。</p> 
<hr /> 
<h2>总个结吧&#xff09;</h2> 
<p>&#x1f511; <strong>递归返回值是最终答案 → 直接 return</strong></p> 
<p>&#x1f511; <strong>递归返回值只是判断材料 → 必须先接住再处理</strong></p>
