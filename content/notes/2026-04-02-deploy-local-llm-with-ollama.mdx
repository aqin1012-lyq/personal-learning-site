<h2>6️⃣ 用 Docker 启动 Open WebUI</h2> 
<pre><code>docker run -d \
  -p 3000:8080 \
  -e OLLAMA_BASE_URL&#61;http://host.docker.internal:11434 \
  -v open-webui:/app/backend/data \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main</code></pre> 
<p>第一次执行本地没有会进行下载&#xff0c;可能会比较慢&#xff0c;等结束即可</p> 
 
<p>
<p><img src="/images/notes/2026-04-02-deploy-local-llm-with-ollama/09.png" alt="2026-04-02-deploy-local-llm-with-ollama-9" /></p>
</p>

<p>&#x1f449; 现在你已经有一个 <strong>完全本地的 ChatGPT 网页版</strong></p> 
<p>可以通过以下几种方式确认使用的是否是本地大模型&#xff1a;</p> 
<p>✅ 看端口是谁在占用</p> 
<p><code>lsof -nP -iTCP:11434 -sTCP:LISTEN</code></p> 
<p>你应该能看到类似 <code>ollama</code> 的进程在监听&#xff08;PID 不同没关系&#xff09;。</p> 
<p>✅ Ollama API 自检</p> 
<p><code>curl </code><code>http://localhost:11434/api/tags</code></p> 
<p>能返回 JSON&#xff08;列出模型&#xff09;就说明&#xff1a;<strong>本机 Ollama 服务正常</strong>。</p> 
<p>✅ 看你的 Open WebUI 容器环境变量是否正确</p> 
<p><code>docker inspect open-webui | grep -n &#34;OLLAMA_BASE_URL&#34;</code></p> 
<p>你应该看到&#xff1a;</p> 
<p><code>OLLAMA_BASE_URL&#61;</code><code>http://host.docker.internal:11434</code></p> 
<p>如果没看到&#xff0c;说明 WebUI 可能没按我们那条命令启动&#xff0c;需要重启容器&#xff08;第 4 步给你一键修复命令&#xff09;。</p> 
<p>✅ 直接在容器里 curl 到你的本机 Ollama</p> 
<p><code>docker exec -it open-webui sh -lc &#34;curl -s </code><code>http://host.docker.internal:11434/api/tags</code><code> | head&#34;</code></p> 
<p>能返回内容就说明&#xff1a;<strong>WebUI 确实能访问你本机 Ollama</strong>。</p> 
 
<p>
<p><img src="/images/notes/2026-04-02-deploy-local-llm-with-ollama/14.png" alt="2026-04-02-deploy-local-llm-with-ollama-14" /></p>
</p> 
<h2>♻️ 重新启动</h2> 
<pre><code>docker stop open-webui
docker rm open-webui
docker volume rm open-webui</code></pre>
