<p>刷二叉树题的时候&#xff0c;我一开始也有个错觉&#xff1a;</p>

<p>不就是递归吗&#xff1f;<code>left</code>、<code>right</code> 一套模板走天下。</p>

<p>直到我在 <strong>minDepth</strong>、<strong>isSymmetric</strong> 这些题上反复 WA&#xff0c;才意识到&#xff1a;<br />
<strong>真正难的不是递归&#xff0c;而是“你在递归里比较的到底是什么”。</strong></p>

<p>这篇文章&#xff0c;我想把二叉树里最容易混的四道经典题&#xff0c;一次性讲清楚&#xff1a;</p>

<ul><li>最大深度 <code>maxDepth</code></li><li>最小深度 <code>minDepth</code></li><li>是否平衡 <code>isBalanced</code></li><li>是否对称 <code>isSymmetric</code></li></ul>

<p>不讲抽象概念&#xff0c;只讲<strong>为什么会错、错在哪、怎么一眼看出来</strong>。</p>

<hr />
<h2>一、maxDepth&#xff1a;最“老实”的一题</h2>

<h2>题意</h2>

<p>返回二叉树的最大深度。</p>

<h2>正确代码</h2>

<pre><code>
<code>int maxDepth(TreeNode root) {
    if (root &#61;&#61; null) return 0;
    return Math.max(maxDepth(root.left), maxDepth(root.right)) &#43; 1;
}</code></code></pre>

<h2>为什么这题几乎不会错&#xff1f;</h2>

<p>因为它的逻辑非常“宽松”&#xff1a;</p>

<ul><li>左边走得深 ✔</li><li>右边走得深 ✔</li><li>只要有一条路深&#xff0c;就算数</li></ul>

<p>哪怕一边是 <code>null</code>&#xff0c;返回 0&#xff0c;也不会影响最终结果。</p>

<p>&#x1f449; <strong>maxDepth 的特点&#xff1a;所有路径都是合法路径。</strong></p>

<hr />
<h2>二、minDepth&#xff1a;第一个真正的坑</h2>

<h2>很多人一开始会这么写&#xff08;比如我。。&#xff09;</h2>

<pre>
<code>int minDepth(TreeNode root) {
    if (root &#61;&#61; null) return 0;
    return Math.min(minDepth(root.left), minDepth(root.right)) &#43; 1;
}</code></pre>

<p>这段代码&#xff0c;看起来和 <code>maxDepth</code> 只差一个 <code>min</code>&#xff0c;但它是 <strong>错的</strong>。</p>

<hr />
<h2>最小反例&#xff08;一定要记住&#xff09;</h2>

<pre>
<code>    1
   /
  2</code></pre>

<p>正确答案&#xff1a;<strong>2</strong></p>

<p>但上面的代码会算成&#xff1a;</p>

<pre>
<code>min(1, 0) &#43; 1 &#61; 1</code></pre>

<p>为什么&#xff1f;</p>

<p>因为你把 <strong>不存在的右子树&#xff08;null&#xff09;</strong> 当成了一条“更短的路径”。</p>

<hr />
<h2>本质原因</h2>

<p><strong>最小深度&#xff0c;要求的是&#xff1a;到“叶子节点”的最短路径</strong></p>

<p><code>null</code> 节点不是叶子节点。</p>

<hr />
<h2>正确写法</h2>

<pre>
<code>int minDepth(TreeNode root) {
    if (root &#61;&#61; null) return 0;

    if (root.left &#61;&#61; null) {
        return minDepth(root.right) &#43; 1;
    }
    if (root.right &#61;&#61; null) {
        return minDepth(root.left) &#43; 1;
    }

    return Math.min(minDepth(root.left), minDepth(root.right)) &#43; 1;
}</code></pre>

<hr />
<h2>三、isBalanced&#xff1a;不是“算高度”&#xff0c;而是“边算边判断”</h2>

<h2>错误但常见的思路</h2>

<ul><li>写一个 <code>depth()</code></li><li>每个节点算左右高度</li><li>再判断差值</li></ul>

<p>这样写会导致<strong>大量重复计算</strong>&#xff0c;性能会退化到 O(n²)。</p>

<hr />
<h2>正确思路&#xff1a;用高度返回值“顺便”传递状态</h2>

<pre>
<code>boolean isBalanced(TreeNode root) {
    return height(root) !&#61; -1;
}

int height(TreeNode root) {
    if (root &#61;&#61; null) return 0;

    int left &#61; height(root.left);
    if (left &#61;&#61; -1) return -1;

    int right &#61; height(root.right);
    if (right &#61;&#61; -1) return -1;

    if (Math.abs(left - right) &gt; 1) return -1;

    return Math.max(left, right) &#43; 1;
}</code></pre>

<h2>这里的关键点</h2>

<ul><li><code>-1</code> 表示“已经不平衡”</li><li>一旦发现不平衡&#xff0c;直接向上传播</li><li><strong>不用再算高度了&#xff0c;提前剪枝</strong></li></ul>

<p>&#x1f449; 这是一个非常典型的“递归返回多重信息”的例子。</p>

<hr />
<h2>四、isSymmetric&#xff1a;最容易“逻辑方向写反”的题</h2>

<h2>常见误区</h2>

<p>很多人&#xff08;比如我一开始&#xff09;会写成&#xff1a;</p>

<pre>
<code>traversal(root.left) &amp;&amp; traversal(root.right)</code></pre>

<p>这其实是在判断&#xff1a;</p>

<p>左子树自己对称<br />
右子树自己对称</p>

<p>但题目要的是&#xff1a;</p>

<p><strong>左子树和右子树是否互为镜像</strong></p>

<hr />
<h2>反例直接说明问题</h2>

<pre>
<code>    1
   / \
  2   2
   \   \
    3   3</code></pre>

<p>这棵树 <strong>不是对称的</strong>&#xff0c;<br />
但“左右各自检查”的写法很容易被它骗过去。</p>

<hr />
<h2>正确写法&#xff1a;一次递归比较两个节点</h2>

<pre>
<code>boolean isSymmetric(TreeNode root) {
    if (root &#61;&#61; null) return true;
    return mirror(root.left, root.right);
}

boolean mirror(TreeNode l, TreeNode r) {
    if (l &#61;&#61; null &amp;&amp; r &#61;&#61; null) return true;
    if (l &#61;&#61; null || r &#61;&#61; null) return false;
    if (l.val !&#61; r.val) return false;

    return mirror(l.left, r.right)
        &amp;&amp; mirror(l.right, r.left);
}</code></pre>

<hr />
<h2>最后一点个人经验</h2>

<p>二叉树递归&#xff0c;真正的难点从来不是代码&#xff0c;而是这三个问题&#xff1a;</p>

<ol><li><strong>空节点代表什么&#xff1f;</strong></li><li><strong>哪些路径是“合法路径”&#xff1f;</strong></li><li><strong>递归函数到底在比较谁跟谁&#xff1f;</strong></li></ol>
